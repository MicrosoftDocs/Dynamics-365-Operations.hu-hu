## <a name="atp-calculations"></a><span data-ttu-id="9797a-116">Ígérethez rendelkezésre áll számítások</span><span class="sxs-lookup"><span data-stu-id="9797a-116">ATP calculations</span></span>
<span data-ttu-id="9797a-117">Az ígérethez rendelkezésre álló mennyiséget a rendszer a „kumulatív és előretekintéssel meghatározott, ígérethez rendelkezésre álló mennyiség” módszerrel számítja ki.</span><span class="sxs-lookup"><span data-stu-id="9797a-117">The ATP quantity is calculated by using the “cumulative ATP with look-ahead” method.</span></span> <span data-ttu-id="9797a-118">Ennek az ígérethez rendelkezésre áll számítási módszernek a fő előnye, hogy kezelhetők olyan esetek, ahol a kiadások összeg a bevételek között nagyobb, mint a legfrissebb bevétel (például amikor egy korábbi bevételezési mennyiséget fel kell használni a követelmények miatt).</span><span class="sxs-lookup"><span data-stu-id="9797a-118">The main advantage of this ATP calculation method is that it can handle cases where the sum of issues among receipts is more than the latest receipt (for example, when a quantity from an earlier receipt must be used to meet a requirement).</span></span> <span data-ttu-id="9797a-119">A „kumulatív ígérethez rendelkezésre állás és előretekintés” számítási módszer minden kiadást magába foglal, amíg a bevételezendő kumulatív mennyiség meghaladja a kumulatív kiadandó mennyiséget.</span><span class="sxs-lookup"><span data-stu-id="9797a-119">The “cumulative ATP with look-ahead” calculation method includes all issues until the cumulative quantity to receive exceeds the cumulative quantity to issue.</span></span> <span data-ttu-id="9797a-120">Ezért az ígérethez rendelkezésre állás számítás módszer kiértékeli, hogy a mennyiség egy része a korábbi időszakból használható-e a későbbi időszakban.</span><span class="sxs-lookup"><span data-stu-id="9797a-120">Therefore, this ATP calculation method evaluates whether some of the quantity from an earlier period can be used in a later period.</span></span>  

<span data-ttu-id="9797a-121">Az Ígérethez rendelkezésre álló mennyiség készletegyenlege nem véglegesített az első időszakban.</span><span class="sxs-lookup"><span data-stu-id="9797a-121">The ATP quantity is the uncommitted inventory balance in the first period.</span></span> <span data-ttu-id="9797a-122">Általában az egyes időszakokra kerül kiszámításra, amelyekben a bevételezés ütemezve van.</span><span class="sxs-lookup"><span data-stu-id="9797a-122">Typically, it's calculated for each period in which a receipt is scheduled.</span></span> <span data-ttu-id="9797a-123">A program kiszámítja az ígérethez rendelkezésre álló mennyiség időszakának napjait, és a mennyiség első napjaként az aktuális dátumot határozza meg.</span><span class="sxs-lookup"><span data-stu-id="9797a-123">The program calculates the ATP period in days and calculates the current date as the first date for the ATP quantity.</span></span> <span data-ttu-id="9797a-124">Az első időszakban az ígérethez rendelkezésre álló mennyiség az aktuális készlet esedékes és lejárt vevői rendelésekkel csökkentett mennyisége.</span><span class="sxs-lookup"><span data-stu-id="9797a-124">In the first period, ATP includes on-hand inventory minus customer orders that are due and overdue.</span></span>  

<span data-ttu-id="9797a-125">A rendszer a következő képlettel számítja ki az ígérethez rendelkezésre álló mennyiséget:</span><span class="sxs-lookup"><span data-stu-id="9797a-125">ATP is calculated by using the following formula:</span></span>  

<span data-ttu-id="9797a-126">Ígérethez rendelkezésre áll = Az előző időszakban ígérethez rendelkezésre álló készlet + Az aktuális időszak bevételezései - Az aktuális időszak kiadásai – Az összes jövőbeni időszak bevételezéseinek összegének időszakáig, minden egyes jövőbeni időszak nettó kiadási mennyisége, a jövőbeni időszakot beleértve és azzal bezárólag nagyobb, mint a kiadások összege a jövőbeni időszakkal bezárólag.</span><span class="sxs-lookup"><span data-stu-id="9797a-126">ATP = ATP for the previous period + Receipts for the current period – Issues for the current period – Net issue quantity for each future period until the period when the sum of receipts for all future periods, up to and including the future period, exceeds the sum of issues up to and including the future period.</span></span>  

<span data-ttu-id="9797a-127">Ha nincs több figyelembe vehető kiadás vagy bevételezés, akkor az ígérethez rendelkezésre álló mennyiség a további dátumokon ugyanaz lesz, mint az ígérethez rendelkezésre álló utolsó számított mennyiség.</span><span class="sxs-lookup"><span data-stu-id="9797a-127">When there are no more issues or receipts to consider, the ATP quantity for the following dates is the same as the latest calculated ATP quantity.</span></span>  

<span data-ttu-id="9797a-128">Ha az ígérethez rendelkezésre álló mennyiség ellenőrzésekor egy cikknek nincs minden dimenziója megadva, a hiányzó dimenziók később is megadhatók a bevételezésekben és kiadásokban.</span><span class="sxs-lookup"><span data-stu-id="9797a-128">If not all the dimensions that are used for an item are given when the ATP check is completed, they can still be specified on the issue and receipts.</span></span> <span data-ttu-id="9797a-129">Ebben az esetben az ígérethez rendelkezésre álló mennyiség számításakor a bevételezéseket és kiadásokat összesíteni kell a meglévő dimenziókban, hogy a mennyiség számításában résztvevő bevételezési és kiadási sorok száma csökkenjen.</span><span class="sxs-lookup"><span data-stu-id="9797a-129">In this case, in the ATP calculation, the receipts and issues must be aggregated to the existing dimensions to reduce the number of receipt and issue lines that are used in the ATP calculation.</span></span>  

<span data-ttu-id="9797a-130">A megjelenített ígérethez rendelkezésre állási mennyiség mindig 0 (nulla) vagy nagyobb.</span><span class="sxs-lookup"><span data-stu-id="9797a-130">The ATP quantity that is shown is always greater than or equal to 0 (zero).</span></span> <span data-ttu-id="9797a-131">Ha az ígérethez rendelkezésre állási mennyiség számítás eredménye negatív (pl. ha az előzetesen ígért mennyiség túllépi az elérhető mennyiséget), akkor a program automatikusan **0** értékre állítja.</span><span class="sxs-lookup"><span data-stu-id="9797a-131">If the calculation returns a negative ATP quantity (for example, if the quantity that was previously promised exceeds the available quantity), the program automatically sets the quantity to **0**.</span></span>
