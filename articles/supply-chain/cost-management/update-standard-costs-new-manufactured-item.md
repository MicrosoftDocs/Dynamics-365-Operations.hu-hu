[!include [banner](../includes/banner.md)]

<span data-ttu-id="d65ed-104">Ez a leírás az újonnan gyártott cikkek alapköltségeinek frissítéséhez nyújt útmutatást.</span><span class="sxs-lookup"><span data-stu-id="d65ed-104">This article provides guidance for updating standard costs for a new manufactured item.</span></span> 

<span data-ttu-id="d65ed-105">Az alábbi irányelvek feltételezik, hogy a kétverziós megközelítés segítségével frissíti az elszámoló költséget.</span><span class="sxs-lookup"><span data-stu-id="d65ed-105">The following guidelines assume that you use a two-version approach to update standard costs.</span></span> <span data-ttu-id="d65ed-106">Ebben a megközelítésben az egyik költségszámítási verzió az eredetileg a befagyasztott időszakra meghatározott elszámolási árakat tartalmazza, míg a másik költségszámítási verzió az újonnan gyártott cikkekhez tartozó járulékos frissítéseket tartalmazza.</span><span class="sxs-lookup"><span data-stu-id="d65ed-106">In this approach, one costing version contains the standard costs that were originally defined for the frozen period, and the second costing version contains the incremental updates that pertain to the new manufactured items.</span></span> <span data-ttu-id="d65ed-107">A járulékos frissítéseket költségrekordként lehet megadni a második költségszámítási verzióban, majd ezután válnak elérhetővé.</span><span class="sxs-lookup"><span data-stu-id="d65ed-107">The incremental updates are entered as cost records in the second costing version, and eventually they are enabled.</span></span> <span data-ttu-id="d65ed-108">A kétverziós megközelítés egy második költségszámítási verzió meghatározását igényli.</span><span class="sxs-lookup"><span data-stu-id="d65ed-108">The two-version approach requires that you define a second costing version.</span></span> <span data-ttu-id="d65ed-109">Az ennek a költségszámítási verziónak a meghatározására vonatkozó irányelvek a következők:</span><span class="sxs-lookup"><span data-stu-id="d65ed-109">Here are the guidelines for defining this costing version:</span></span>

-   <span data-ttu-id="d65ed-110">Rendeljen hozzá egy **Alapköltség** típusú költségszámítást.</span><span class="sxs-lookup"><span data-stu-id="d65ed-110">Assign a costing type of **Standard cost**.</span></span>
-   <span data-ttu-id="d65ed-111">Rendeljen hozzá egy egyértelmű azonosítót a költségverzióhoz, amely utal a költségverzió tartalmára. Például: **2016-FRISSÍTÉSEK**.</span><span class="sxs-lookup"><span data-stu-id="d65ed-111">Assign a significant identifier that indicates the contents of the costing version, such as **2016-UPDATES**.</span></span>
-   <span data-ttu-id="d65ed-112">Az **Ártípusok engedélyezése** mezőcsoportban, győződjön meg róla, hogy az **Önköltségi ár** értéke **Igen**.</span><span class="sxs-lookup"><span data-stu-id="d65ed-112">In the **Allow price types** field group, make sure that **Cost price** is set to **Yes**.</span></span>
-   <span data-ttu-id="d65ed-113">Engedélyezze a költség feljegyzések bevitelét minden webhely számára úgy, hogy üresen hagyja a **Webhely**mezőt.</span><span class="sxs-lookup"><span data-stu-id="d65ed-113">Allow cost records to be entered for all sites (that is, leave the **Site** field blank).</span></span> <span data-ttu-id="d65ed-114">Ha megad egy webhelyet, akkor a költségrekordok csak arra a webhelyre rögzíthetők.</span><span class="sxs-lookup"><span data-stu-id="d65ed-114">If you enter a site, cost records can be entered only for that site.</span></span>

2.  <span data-ttu-id="d65ed-121">Használja a **Cikk ár** oldalt, hogy rögzítse az új beszerzett cikkek költségrekordjait.</span><span class="sxs-lookup"><span data-stu-id="d65ed-121">Use the **Item price** page to enter cost records for new purchased items.</span></span> <span data-ttu-id="d65ed-122">Minden költségrekordnál adja meg a költségszámítási verziót, amely tartalmazza a járulékos frissítéseket, és adjon meg olyan kezdő dátumot, amely megelőzi az új cikkek várható gyártásának a dátumát.</span><span class="sxs-lookup"><span data-stu-id="d65ed-122">For each cost record, enter the costing version that contains incremental updates, and use a from-date that comes before the expected manufacturing date for new manufactured items.</span></span>
3.  <span data-ttu-id="d65ed-123">Számítsa ki az új gyártott cikkek költségét a **Kiszámítás** lap segítségével.</span><span class="sxs-lookup"><span data-stu-id="d65ed-123">Calculate the cost of new manufactured items by using the **Calculation** page.</span></span> <span data-ttu-id="d65ed-124">Nyissa meg a **Kiszámítás** lapot a **költségszámítási verzió fenntartása** oldalról, majd válassza ki kezdik.</span><span class="sxs-lookup"><span data-stu-id="d65ed-127">Enter a from-date for the bill of materials (BOM) calculation that corresponds to the start of manufacturing for the new data-ttu-id="d65ed-129">**Megjegyzés:** A hiányzó költségrekord jelezheti az újonnan gyártott cikket.</span><span class="sxs-lookup"><span data-stu-id="d65ed-129">**Note:** A missing cost record can indicate a new manufactured item.</span></span> <span data-ttu-id="d65ed-130">Az Anyagjegyzék-számítások korlátozhatók a hiányos költségekkel rendelkező cikkekre.</span><span class="sxs-lookup"><span data-stu-id="d65ed-130">BOM calculations can be limited to items that have missing costs.</span></span>

5.  <span data-ttu-id="d65ed-134">Használja a **Költségszámítási verzió beállítása** lapot, hogy módosítsa a zárolás jelzőt, ezzel lehetővé téve a második költségszámítási verzióba foglalt függőben lévő költségrekordok aktiválását.</span><span class="sxs-lookup"><span data-stu-id="d65ed-134">Use the **Costing version setup** page to change the blocking flag to allow activation of the pending cost records in the second costing version.</span></span>
