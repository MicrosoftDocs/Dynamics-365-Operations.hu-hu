### <a name="preset-and-custom-layouts"></a><span data-ttu-id="eca7b-175">Előre beállított és egyéni elrendezések</span><span class="sxs-lookup"><span data-stu-id="eca7b-175">Preset and custom layouts</span></span>

<span data-ttu-id="eca7b-176">A webhely elrendezései lehetnek *előre beállítottak* vagy *egyéniek* is:</span><span class="sxs-lookup"><span data-stu-id="eca7b-176">Layouts on your site can be either *preset* or *custom*:</span></span>

- <span data-ttu-id="eca7b-181">Az **Egyéni elrendezések** lényegében egyszer használatos elrendezések, amelyeket egyetlen lapra ágyaznak be.</span><span class="sxs-lookup"><span <span data-ttu-id="eca7b-184">Ezt követően, ha a szerző újra fel szeretné használni az elrendezést más oldalakra, akkor egyszerűen konvertálható előre beállított elrendezéssé.</span><span class="sxs-lookup"><span data-stu-id="eca7b-184">Then, if the author wants to reuse the layout for other pages, it can easily be converted to a preset layout.</span></span> <span data-ttu-id="eca7b-185">Az új előre beállított elrendezést a program az ugyanabból a sablon-hierarchiából származó lapok esetében, valamint az elrendezési váltási forgatókönyvekben beállításként from the same template hierarchy.</span></span> <span data-ttu-id="eca7b-186">Ez fordítva is működik, tehát az előre beállított elrendezések leágaztathatók egyéni elrendezésekhez.</span><span class="sxs-lookup"><span data-stu-id="eca7b-186">Conversely, preset layouts can be branched into custom layouts.</span></span> <span data-ttu-id="eca7b-187">Ily módon a szerző leválaszthatja a lapot <span data-ttu-id="eca7b-189">Az előre beállított elrendezések és az egyéni elrendezések a szerkesztési eszköztár különböző részein szerkeszthetők.</span><span class="sxs-lookup"><span data-stu-id="eca7b-189">Preset layout and custom layouts are edited in different parts of the authoring toolset.</span></span> <span data-ttu-id="eca7b-190">Mivel az egyéni elrendezések nem rendelkeznek függőségekkel más lapokon, közvetlenül a lap szerkesztőjében szerkeszthetők.</span><span class="sxs-lookup"><span data-stu-id="eca7b-190">Because custom layouts have no dependencies on other pages, they are edited directly in the page editor.</span></span> <span data-ttu-id="eca7b-191">Ebben az esetben transzparens leginkább az elrendezés megléte a felhasználó számára, és csak a lapszintű tulajdonságokban és az elrendezési beállításokon keresztül érhető el.</span><span class="sxs-lookup"><span data-stu-id="eca7b-191">In this case, the existence of a layout is mostly transparent to the user and is exposed only in page-level properties and through the actions for layout options.</span></span> <span data-ttu-id="eca7b-192">Mivel azonban az előre beállított elrendezések módosításai hatással lehetnek a származtatott elemekre, az elrendezésszerkesztőben kell azokat szerkeszteni, ahol a közzétételi műveletek figyelembe veszik a lefelé irányuló, a származtatott oldalakra gyakorolt hatásokat.</span><span class="sxs-lookup"><span data-stu-id="eca7b-192">However, because changes to preset layouts can affect many child pages, they must be edited in the layout editor, where publish actions consider the full downstream impact on child pages.</span></span>
