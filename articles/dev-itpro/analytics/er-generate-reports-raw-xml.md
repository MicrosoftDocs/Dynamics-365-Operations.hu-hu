# <a name="generate-reports-by-adding-content-as-raw-xml"></a><span data-ttu-id="4bb35-103">Jelentések készítése tartalom nyers XML-kódként való hozzáadásával</span><span class="sxs-lookup"><span data-stu-id="4bb35-103">Generate reports by adding content as raw XML</span></span>

[!include[banner](../includes/banner.md)]

<span data-ttu-id="4bb35-104">Használhatja az új **Nyers XML** formátum elemet elektronikus jelentési (ER) formátumok kimenő dokumentumok XML-formátumú előállításához.</span><span class="sxs-lookup"><span data-stu-id="4bb35-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="4bb35-105">Bizonyos esetekben előfordulhat, hogy szeretné nyers XML-adatok hozzáadása ezeket a jelentéseket, a következő okok valamelyikéből:</span><span class="sxs-lookup"><span data-stu-id="4bb35-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="4bb35-106">A nyers XML használata kényelmesebb a jelentés eredeti tervezésére és karbantartásához, mert az XML-szerkezetet automatikusan létre lehet hozni egy futásidejű kifejezés végrehajtásával.</span><span class="sxs-lookup"><span data-stu-id="4bb35-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="4bb35-107">Többszörös kötéseket emiatt nem kell a tervezéskor több formátumelemhez meghatározni.</span><span class="sxs-lookup"><span data-stu-id="4bb35-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="4bb35-108">Akkor lehetséges, ha az éppen használt adatforrások tartalmaznak információkat, amelyek a jelentés előállításakor XML-elemek létrehozásához használhatók.</span><span class="sxs-lookup"><span data-stu-id="4bb35-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span>
- <span data-ttu-id="4bb35-109">Nincs semmilyen más módszer, amellyel korábban már kapott, és a rendszerben tárolt XML-tartalommal tölthető lenne fel a jelentés.</span><span class="sxs-lookup"><span data-stu-id="4bb35-109">No other method can be used to fill the report with XML content that was previously received and stored in the system.</span></span> <span data-ttu-id="4bb35-110">Például a létrehozott XML-válasznak előfordulhat, hogy tartalmaznia kell a korábban elküldött XML-kérés tartalmát.</span><span class="sxs-lookup"><span data-stu-id="4bb35-110">For example, the XML response that is generated might have to contain the content of an XML request that was sent earlier.</span></span>
- <span data-ttu-id="4bb35-111">Nincs más módszer, amelynek a használatával lehetne karaktereket beszúrni a generált dokumentumba a numerikus kódjuk alapján.</span><span class="sxs-lookup"><span data-stu-id="4bb35-111">No other method can be used to insert characters into the generated document based on their numeric codes.</span></span> <span data-ttu-id="4bb35-112">Bizonyos nyelvek és karakterek esetében az ilyen típus nem létezik.</span><span class="sxs-lookup"><span data-stu-id="4bb35-112">For some languages and characters, codes of this type don't exist.</span></span> <span data-ttu-id="4bb35-113">Példák a görög betű rhó (ρ) és a HTML-entitás kódok, például \&eacute; az egy *e*, amelynek ékezete van (é).</span><span class="sxs-lookup"><span data-stu-id="4bb35-113">Examples include the Greek letter rho (ρ) and HTML entity codes such as \&eacute; for an *e* that has an acute accent (é).</span></span>

> [!NOTE]
> <span data-ttu-id="4bb35-114">Ne feledje, hogy a keretrendszer nem ellenőrzi, hogy **Nyers XML** formátum segítségével a generált dokumentumba kerülő XML-tartalom helyes-e.</span><span class="sxs-lookup"><span data-stu-id="4bb35-114">Be aware that the framework doesn't control whether the XML content that is placed to the generated document by using the **RAW XML** format element is correct.</span></span>
